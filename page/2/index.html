建造者模式

2022-03-31

定义与类型

  1. 定义:将一个复杂对象的构建与他的表示分离，使得同样的构建过程可以创建不同的表示
  2. 用户只需要指定需要构造的类型就可以得到他们，建造过程及细节不需要知道
  3. 就是如何一步一步建造一个包含多个组件的对象，相同的构建过程可以创建不同的产品，

适用场景

  1. 如果一个对象有非常复杂的内部结构（很多属性）
  2. 并且想把复杂对象的创建和适用分离

优点

  1. 封装性好，创建和适用分离
  2. 扩展性好，建造类之间独立，一定程度上解耦

缺点

  1. 产生多余的Builder对象
  2. 产品内部发生变化，建造者都需要修改，成本较大

和工厂模式比较

  1. 建造者模式更注重于方法的调用顺序，而工厂模式注重于创建产品
  2. 创建对象的力度不同，建造者模式可以创建复杂的产品，由各种复杂的部件组成，工厂模式创建出来的都是一个样子
  3. 关注点不同，工厂模式注重把这个对象创建出来就ok了，而建造者模式，不止要创建出这个产品，还要知道这个产品都是由哪些部件组成的
  4. 建造者模式再某些场景，比如，一定的顺序决定了产出的产品不一样，这里也要对顺序进行跳转，但是工厂并不关心顺序
  5. 当创建一个对象需要很多步骤时，适合适用建造者模式，而通过一个简单的方法就要创建出一个简单的对象时，适合适用工厂相关模式

Netty

  • netty是一个提供了易于使用的API的客户端/服务器框架

  • 并发高-NIO

  • 传输快-零拷贝

    有一些数据要从IO里面读取，然后放到堆里面，中间会经过一些缓冲区，先把数据读出来，放到缓冲区，然后再从缓存区读出来，放到堆里面，这样数据会拷贝两次才能进入堆内存中

    Netty使用了NIO中的零拷贝的一个特性，当他去传输数据的时候，会开辟一个新的堆内存，数据直接从IO读到了新开辟的内存中，这样就加快了数据传输的一个速度

阻塞与非阻塞

  • 线程访问资源，该资源是否准备就绪的一种处理方式
  • 阻塞就是一直等待这个资源处理完毕，在此期间，不可以做任何的事情
  • 非阻塞是指线程直接返回一个结果，不会等待资源处理完之后才响应，他会去请求别的资源

同步与异步

  • 同步和异步是指访问数据的一种机制
  • 同步是指主动请求，并且会等待IO，操作完成之后，他会有一个通知，会通知到你，他是一个同步的
  • 异步是指当我们一个线程发送请求之后，他可以继续处理其他的任务，发起其他的请求，当我们很多请求发送完毕之后，他会逐一的通知你

BIO NIO AIO

  • BIO 同步阻塞IO， Block IO，IO再进行读写的过程中，这个线程是阻塞的，不能做其他的事情
  • NIO
    1. 同步非阻塞IO，New IO（Non-Block IO）IO 在读写的过程中，如果资源被占用，他会去做其他的事情，过会儿再来看这个资源能不能使用了。
    2. 有三个客户端，一个服务器，他们再通信的时候会使用到一个Selector多路复用器，是一个单线程，客户端增多不会影响性能，客户端连接服务器时，会先再Selector上进行注册，注册完成后会生成一个channel1，这个channel是一个双向通道，可以进行一些数据的读写，这些读写都会进入缓存区里面去，是一种非阻塞的读取，如果没有数据，Selectot会直接跳过，不会一直在哪里等着你，buffer里面的数据会进行一个读写，数据被读完之后，还会存在buffer里面的，不会像stream一样读完之后就消失了
  • AIO
    1. 异步阻塞IO ，这种方式很少用，如果资源被占用了，我会一直等着，知道资源释放后通知我，我再使用，非常傻
    2. 异步非阻塞IO AIO ，如果资源被占用了，我不会等着，我会做其他的事情，等资源释放后通知我，然后我再去使用该资源

Reactor线程模型

  • 单线程模型: 所有的IO操作都由同一个NIO线程处理的，连接的时候是这个单线程处理，连接完之后还是这个单线程来处理，作为服务端，来接收客户端所有的连接，作为客户端，就会发起所有的连接，异步非阻塞IO，理论上是没有问题的
  • 多线程模型：由一组NIO线程处理IO操作，将原先的单线程拆分成了两块，一块是Reactor单线程，一块是Reactor线程池，单线程来连接，连接之后其余的操作交给线程池里面的线程来搞
  • 主从线程模型：由一组的线程池来接收请求，由一组线程池处理IO

Hello Netty 服务器

  • 定义服务器启动类

  • 为服务器设置Channel 当我们客户端和服务器建立连接之后，就会有一个双向的通道，这个通道就是channel，需要定义一个Channel的类型

  • 设置处理线程池的助手类初始化器，会有很多助手类对channel进行处理，比如编解码处理，写数据，读数据，等等，类似于拦截器，配置多个拦截器来拦截channel

    1. 每一个channel由多个Handler共同组成管道pipeline，我们需要为他设置很多Hanlder助手类
    3. 可以将pipeline看成一个拦截器组，当请求过来的时候，我们会逐个处理

    public class HelloServer {

    public static void main(String[] args) throws Exception {


    //当我们有一些客户端连接到服务器之后，会通过这样一个线程组来注册，然后获得相应的channel，然后丢给下面的一个线程组来处理

    //定义一对线程组
    //主线程组  用于接收客户端的连接，但是不做任何处理
    NioEventLoopGroup bossGroup = new NioEventLoopGroup();
    //从线程组  老板线程组会把任务丢给从线程组来做相应的任务
    NioEventLoopGroup workerGroup = new NioEventLoopGroup();


    // Netty 服务器的创建 serverBootstrap是一个启动类
    try {
        ServerBootstrap serverBootstrap = new ServerBootstrap();
        serverBootstrap.group(bossGroup,workerGroup)            //配置线程组
                        .channel(NioServerSocketChannel.class)  //设置NIO的双向通道
                        .childHandler(new HelloServerInitializer());  //针对从线程组做一些操作，助手类

        //首先绑定一个端口，然后设置一个同步的启动方式，会等待8088启动完毕
        ChannelFuture channelFuture = serverBootstrap.bind(8088).sync();

        //监听关闭的channel，设置为同步方式
        channelFuture.channel().closeFuture().sync();
    } finally {
        //优雅的关闭线程池
        bossGroup.shutdownGracefully();
        workerGroup.shutdownGracefully();
    }
}

    }
