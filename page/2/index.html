打印二叉树的后继节点

发表于 2019-12-30 更新于 2020-01-02

中序遍历后，4的后继节点是2，2的后继节点是5 ，。。。。

如果一个节点，假如说X，

如果X有右子树，那么X

的后继节点一定是这个右子树最左的节点

如果X没有右子树，那么向上找，什么时候停。找到当前节点是父节点的左孩子停，那个父节点就是原始节点的后继节点

如果是6，那么我就找，那一颗数的左子树是以6结尾的，就是3嘛

如果是7，那一颗树的左子树是以7结尾的，米有啊，所以7的后继不存在，

通过x的父指针，找到x的父，如果我发现X是这个父节点的左孩子，那不行，没有找到，指针继续指向现在节点的父节点，看看原来的那个节点是不是这个父节点的右孩子。如果是这个节点就是后继节点。

比如说11：

有没有右子树？ 没有。那就找那一颗树的左子树是11.。

先用父指针找到父节点5，看看5的左子树是不是11，不是。

继续向上找5的父节点是2,2的左子树是4，不是2，所以2也不是11的后继节点

继序向上找2的父节点是1,1的左子树是2，找到了，所以11的后继节点就是1；

找前驱节点

如果有左子树，就是左子树最右的节点

如果没有左子树，就向上找，当前节点是父节点的右孩子就停

二叉树的遍布（非递归）

发表于 2019-12-30 更新于 2020-01-03

先序遍历

先序遍历，和BFS很像，用栈实现的。先压右 后压左，所以栈里面从栈顶到栈低的顺序是逆序的，先压的右，后压的左，那么就是先弹左，后弹右。所以就是当前节点在前，左， 右 。

为啥不用队列，这是二叉树遍历，遍历到底需要找父节点，需要向上走，栈结构天然提供了向上走的结构。

中序遍历

1压到栈中

head向左移动

h还不等于null，

head向左移动

会把当前节点的左边界全部压入栈中

一直到4的左孩子是空，这个过程就停了

再次进入这个while的时候，还会继续while 因为虽然head为空了，但是栈不为空，所以就会继续

现在走进else分支

栈中弹出一个节点，然后打印这个节点

head然后向右移动

结束

当前节点为空，从栈中拿一个，打印，然后当前节点向右跑

如果当前节点不为空，当前节点压入栈，当前节点向左跑

后序遍历

左右中，

先序遍历顺序：中左右

改吧改吧就能改成 中 右 左 ，就是先压left 后压right，换一下顺序

然后不打印，放到一个新的栈中，

新的栈弹出时会把他逆序，也就变成了 左右中的顺序了

struts和springMVC的区别

发表于 2019-12-29

struts的springMVC的区别。

（1）springmvc的入口是一个servlet即前端控制器（DispatchServlet），而struts2入口是一个filter过虑器（StrutsPrepareAndExecuteFilter）。

（2）springmvc是基于方法开发(一个url对应一个方法)，请求参数传递到方法的形参，struts2是基于类开发，传递参数是通过类的属性

SpringMVC创键是在该类上添加@Controller注解就可以了。

Struts前台向后台传递数据方式：

表单中的input标签的name属性值 和 超链接中=号左边的单词

  1. 和Action中属性名一致

  2. 封装成POJO，写在类的属性中，如果这样，前台获取值需要类名.属性名

  3. 实现ModelDriven接口，前台就不用类名.属性名了，直接属性名。

    action中的属性都要有getSet方法

SpringMVC：

  1. 只要和请求的方法的形参的参数名相同就可以了，

  2. 还可以使用request.getParametor()来获取

  3. 把前台的数据封装到POJO中，方法的形参写 POJO pojo就行，

    此时前台中得name的值就不是参数名了，而是pojo的属性名。

    如果是复杂pojo ，name的值也可以是pojo的属性名.属性名。

    后台向前台传值

    struts直接操作对应的属性就可以

    springMVC 可以用ModelAndView.addObject(“obj”,obj)来传递

    或者在参数中声明一个Model model ，使用model.addAttribute(“obj”,obj);

跳转到前台页面：

struts用return一个字符串，并在action标签中配置result标签 name= “字符串名” 标签的值是该jsp页面的全路径

springMVC配好视图解析器后直接return”jsp页面的名字” 或者ModelAndView .setViewName(“jsp页面的名字”);
