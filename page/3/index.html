布隆过滤器

黑名单问题

有一个需求，100亿个url是黑名单，

当你想搜索一些东西的时候，我不想显示某一些url ，100亿个url 都是 64字节，

当搜索某一个url的时候，我会返回一个true或者false ，如果这个url在黑名单里，返回true，

如果不在黑单里，返回false

如果用一个hashSet来搞的话，至少需要6400亿个字节（其实远远不止）

也就是640G的内存，如果这样的话，不是很好做

也可以用hash函数分流，放到不同的机器上，但是也会浪费机器

所以我想不用hashSet 用其他的set 来搞，这个就是布隆过滤器

这个有失误率

如果你有一个url 确实属于黑名单，一定会返回true 但是，有时候一个url不是黑名单，也有可能

返回true（宁可错杀一千，不可放过一个）

是某种类型的集合，但是有失误率的。

如果碰到这类题，你可以先用哈希函数分流来搞。浪费空间。

问：允许有极低的失误率么？可以有

布隆过滤器

是一个比特类型的map

我有一个数组，长度为1000.

这个数组每一个位置上是一个32bit

一个int是4个字节，32位比特。

假如我要把第30000个bit描黑。我需要

30000/32 得到第几个桶。937

30000%32 得到这个桶的第几个bit应该被描黑 ，16

第937个桶里第16位应该被描黑

1 左移16位得到 第16位为1

然后和arr[937] | 1<<16 . 因为arr[937] 初始值是00000000，所以做或运算后该位置就被标记了

url 根据一个hash函数算出一个hash值，这个hash值再去取模m，这样得到0到m-1位置上的一个，算出来的位置描黑。

布隆过滤器是准备K个hash函数 ，这个url 经过hash函数 ，url经过第一个hash函数的时候，描黑，经过第二个hash函数的时候，描黑，有可能两个hash函数打到同一个位置，已经描黑的就继续黑下去，这K个hash函数时相互独立的

一个url 经过k个hash函数，对应的K个位置上描黑，这个url 叫做进到布隆过滤器里面了，

接下来，每一个url 都这么算，然后加到这个bit类型的数组里，所以你准备的这个比特类型的数组应该长一些，要不这100亿个url 进行描黑后，就全黑了，查就没有意义了

怎么查？

我现在黑名单加载完了，整个这个bit类型的数组相当多的位置被描黑了

当我查一个url 的时候，怎么知道这个url在不在这个黑名单里？

这个url 经过K个hash函数，算出了K个位置，如果这K个位置都是黑的，我就说，这个url 在这个黑名单里

如果算出的hash函数有一个不是黑的，我就说这个url 不在黑名单里

如果你的数组很小，都被描黑了，在进来的每一个url不都在黑名单里了？ 就没有意义了

所以我可以调整数组的大小和hash 函数的个数来调整失误率。

数据库连接池

概念：平常我们每对数据库操作一次就会创建一个数据库连接，然后对数据库操作，然后在关闭连接。这样极大部分的时间都浪费在获取连接和销毁连接上了，使用数据库连接池，里面放着连接，需要的时候从连接池中获取一个连接，操作完毕后也不需要释放连接，就把连接归还给连接池。这样就提高了效率。

hash表

hash函数：

  1. 输入域是无穷大的
  2. 输出域是固定的
  3. 输入的参数如果相同的话，输出的也是相同的
  4. 当输入不一样，也有可能得到相同的输出值
  5. 假如输入域是0-99，输出域是0-2

​ 离散型，均匀分布的性质。 0 ，1， 2 差不多都是 33 个

输出和输入的规律无关 比如 陈广1 和陈广2 经过hash处理后 输出可能会差别特别大。 而且在整个输出域均匀分布

hash表的细节

有一个数组。比如大小是17.

有一个map key 是 A value 是 2

我先用key得到一个hash值，然后在对17取模就可以得到一个数字 假设是2。

我先看看 2 的位置有没有元素，没有。就直接挂在 2的位置

假设 还有一个map key 是A value 是 3；

去数组还是找2的位置。已经有元素的是A ，比对value的值， 如果不同，替换原来value的值，相当于修改操作。

如果这个数组里面没有A ，就加上一个A ,value 是3；

就是如果冲突了，就往上挂，一个链表而已。

数组上每个位置的元素都是均匀涨的。

比说我某个链的长度是3了，那么其它链的长度也差不多是3了

此时，我就进行hash表的扩容

为什么扩容呢？因为我觉得这个链表的长度已经是5了。在多的话会影响效率。

原来的数组大小是17.现在假设扩容后是104.

以前都是对17取模，现在对104取模。原来的就失效了。

所以就要把每个数据取出来重新用hash函数算一下，算完之后对104取模。重新分配他在哪一个桶里。

为什么能到达O（1）呢？

如果每次扩容的时候，都让长度扩大一倍，到n的时候要扩 logn的复杂度。问题是 我可以把容量扩大5倍，这样效率又要快。但还不是O（1）

实际上，还可以进行离线扩容

当链表已经到5了，我用的时候还用这个hash表，我在离线的时候进行扩容，当扩容结束后，在把所有的请求打到新的hash表上

因为有很多的优化技巧，所以很快，可以说是O（1）的

JVM中还是有一个桶，每一个桶里放一颗红黑树，是一颗平衡搜索二叉树。这样做出来的hash表

但是经典的结构是桶里面放链表

有一个大文件，每一行都有一个字符串，我想把重复的字符串打印出来

问：你给我多少台机器啊？

答：1000台

你把机器编号，从0 到999

问：文件存在哪里啊？

答：存在一个分布式系统上

问：我想处理这个大文件，按行读，又没有高效的工具啊？

答: 有，你就告诉我怎么处理把

我用这个高效的工具把每一行的数据读出来文本

利用hash函数生成一个hashCode

用hashCode对1000取模

判断是否是完全二叉树

完全二叉树是 一颗二叉树 满足 上面的层都是满的，新的层都是从左向右的。

评判标准1：有右孩子没有左孩子，就不行

我依次拿到的1，2，3，4，5，6，7，8，9，在这个过程中任何一个节点，有右孩子没有左孩子，直接返回false

评判标准2 ：左右两个孩子不全

boolean flag = false;在2之前都为false，满足2的条件后就为true，代表一个阶段，后面遇到的所有节点，都必须是叶子节点

例如：

遇到1，中情况1了没（有右没左）？没有

中情况2了么（左右两个孩子不全）？ 没有

来到2，来到3，来到4，

来到5，中情况1了没（有右没左）？没有

中情况2了么（左右两个孩子不全）？ 中了哈

所以5以后的节点，6，7，8，9，10都必须是叶子节点，否则返回false

打印二叉树的后继节点

中序遍历后，4的后继节点是2，2的后继节点是5 ，。。。。

如果一个节点，假如说X，

如果X有右子树，那么X

的后继节点一定是这个右子树最左的节点

如果X没有右子树，那么向上找，什么时候停。找到当前节点是父节点的左孩子停，那个父节点就是原始节点的后继节点

如果是6，那么我就找，那一颗数的左子树是以6结尾的，就是3嘛

如果是7，那一颗树的左子树是以7结尾的，米有啊，所以7的后继不存在，

通过x的父指针，找到x的父，如果我发现X是这个父节点的左孩子，那不行，没有找到，指针继续指向现在节点的父节点，看看原来的那个节点是不是这个父节点的右孩子。如果是这个节点就是后继节点。

比如说11：

有没有右子树？ 没有。那就找那一颗树的左子树是11.。

先用父指针找到父节点5，看看5的左子树是不是11，不是。

继续向上找5的父节点是2,2的左子树是4，不是2，所以2也不是11的后继节点

继序向上找2的父节点是1,1的左子树是2，找到了，所以11的后继节点就是1；

找前驱节点

如果有左子树，就是左子树最右的节点

如果没有左子树，就向上找，当前节点是父节点的右孩子就停
