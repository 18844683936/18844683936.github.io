栈实现队列

两个栈，push pop

用户添加一个数，永远放到push栈中。

用户拿一个数，永远从pop栈中拿

比如压栈1，2，3，4，5

用户需要拿一个数。就把push栈中所有的东西都给pop栈。顺序就有了

有两个限制：

  1. 如果push栈决定要向pop栈中倒数据，一次性倒完
  2. 如果pop栈中有东西，push栈中一定不能倒数据

倒数据行为可以发生在任意时刻。只要满足这两个限制

实现一个特殊的栈，基本功能上添加返回栈中最小元素的操作

设计：

准备两个栈 ，一个是data栈 一个是min栈

data栈压入一个4， min 也栈压入一个4;

data再压入一个5，min栈：5和min栈栈顶比较，如果5比min栈栈顶(4)小，就压5。如果min栈栈顶（4） 更小，就重复压入4

此时data在压入一个3，min栈：3<4 比min栈栈顶小，就压入3；

如果data栈压入一个6，min栈栈顶3小，重复压入一个3

data栈是一个正常的栈。min栈栈顶就是这个data栈中的最小元素

如果要弹出一个数的时候，data弹一个数，min弹一个数。

用数组结构实现大小固定的队列和栈

实现栈

实现队列

start 变量表示如果我要取一个数，我要把哪一个数取出来给用户

end 变量表示如果我要存一个数，我应该把他填到那个位置。

这两个变量都指向0的位置

单独设置一个size变量来约束start和end

整个数组大小是3

如果size变量没有到3

就把用户新给我的数放到end位置上去 这是end和size的规则

size 同时和start有约束。如果size不等于0

总把start上的数给用户。

所以start和end之间是解耦的。

假设用户添加一个数字7，size = 0 <3 把7放到end的 位置（0） end+1 = 1。size 变成1；

用户让我加6，size = 1 < 3 把6 放到end的位置（1） end+1 = 2 size + 1 = 2；

用户让我返回一个数，我看size是不是0 如果size（2）不是0.就返回 start指的那个数，然后start+1 size-1 = 1；

如果用户要添加一个数字5 ，就把end的位置添加5 size +1 = 2 end+1 =0。end的逻辑是只要end到底了就回到开头 end会跳回到 7 的位置（也就是头一个数0）去。

用户加一个数字9 ，此时 size 为2 还没有道3 ，所以我把9 放到end的位置上。end +1 = 2 ；size +1 = 3；

此时用户再添加一个数，因为size已经为3 了，所以报错；

这是一个循环的队列，end一旦到底就回到开头，start一直在追这个end；

这种方法和之前的队列代码不同，引入了size变量。使得代码思路更加清晰。好写。

排序

关于排序的稳定性： 为什么需要稳定性？因为现实世界中不想要抹去原来的排序的资料

工程中的排序：

                    1. 如果是基本数据类型，就用快排（因为基本数据类型相同的值无差别）
                       2. 如果是引用数据类型，就用归并排序（引用数据类型不同的对象有差别）
                       3. 如果长度小于60，就用插排。因为常数项极少，O（n2）的劣势体现不出来
                       4. 如果长度较大，用的是快排和堆排序，有一个分值的过程。这个过程中如果进行到数组长度小于60的话就直接使用插排了（在递归时有一个判断 如果L>R-60 的话就直接插排）

冒泡排序：可以实现稳点，两个相邻的树比较的时候，前面的和后面的相同，则不交换。continue？

选择排序。：不可以实现稳定性

插入排序：可以实现稳定性

归并排序：可以实现稳点性

快速排序：不可以实现稳点性

partition的过程做不成稳点性。随机选择一个数字。

堆排序：不可以实现稳点性 因为在建立大根堆的过程中就把稳点性破坏了。

堆排序

要说堆排序就得知道堆是什么，堆结构就是一个完全二叉树。

完全二叉树是 一颗二叉树 满足 上面的层都是满的，新的层都是从左向右的。

使用数组来模拟堆结构。由于完全二叉树的性质。可以用数组来模拟堆结构，也就是说，给你一个数组，你给我脑补一颗完全二叉树。

这颗树是你脑海中的，

规则如下：你可以自己画图验证

堆分为两种

  1. 大根堆：在这颗完全二叉树中，任何一颗子树的最大的值都是这颗子树的 头部

  2. 小根堆：任何一颗子树的最小的值都是这颗子树的头部

    堆排序

    1. 让这个数组变成大根堆（未必有序）

    2. 把最后一个位置和堆顶位置交换

    3. 此时可以size N-1

    4. 把剩下的也就是0—-n-1 之间的堆 进行从0开始的heapfiy

    5. 这样就会重新形成一个大根堆

    6. 然后把堆顶和现在最后一个节点做交换

    7. 以此往复，直到我堆的大小减完，最大值放在了最后。

    把一个数组变成大根堆

    总结

    建立堆的过程中

    向上调整。heapInset（）

    当我有一个值变小了，就要往下调整 heapfiy（）

    作用：求中位数

    如何求中位数呢？

    要有一个大根堆，一个小根堆，

    第一个数，进大根堆 5.

    第二个数， 4， 大根堆放4

    发现，大根堆的大小为2，小根堆的大小为0（heapsize会告诉你已经形成堆的大小）发现差值大于1.

    那就在大根堆里弹出一个数到小根堆（此时heapsize-1）

    在进来两个数，6，7 ，全都放到了小根堆上。此时小根堆上的heapsize为3，大根堆的是1，不平衡了

    然后把小根堆的头弹出来，放到大根堆中

为什么大根堆放的是较小的N/2?

策略是当期数如果小于等于大根堆的堆顶，才进大根堆。如果大于大根堆的对顶就进小根堆

大根堆要弹出一个数，是最大的那个数，也就是堆顶

小根堆要弹出一个数，是弹出最小的那个数，放到大根堆中

所以一定是较小的n/2

递归分治的时间复杂度

master公式

T（N） = a*T(N/b) +O(N^d)

  1. log(b,a) > d —-> O(N^ log(b,a))

  2. log(b,a) = d —-> O(N^d * logN)

  3. log(b,a) < d —-> O(N^d)

    a是一共这个过程发生多少次 ，b是子问题的样本量， N是父问题的样本量 d是除去子问题还需要的复杂度

    限制条件：划分子问题的规模是一样的

小和问题和逆序对问题

小和问题

在一个数组中，每一个数左边比当前数小的数累加起来，叫做这个数组的小和。求一个数组 的小和。

例子：

[1,3,4,2,5]

1左边比1小的数，没有；

3左边比3小的数，1；

4左边比4小的数，1、3；

2左边比2小的数，1；

5左边比5小的数，1、3、4、2；

所以小和为1+1+3+1+1+3+4+2=16

这个问题是用归并排序来做的

在每次合并的时候求出小和然后返回。

思路：

两个有序的数组，左边数组中的一个数据X比右边数组中第P个数小。那么由于是有序的数组，P后面的所有数据都比X大。所以 这次的小和就是 X ✖ （ P后面的个数 +1）.
