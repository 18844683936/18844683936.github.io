hash表

发表于 2020-01-05 更新于 2020-01-07

hash函数：

  1. 输入域是无穷大的
  2. 输出域是固定的
  3. 输入的参数如果相同的话，输出的也是相同的
  4. 当输入不一样，也有可能得到相同的输出值
  5. 假如输入域是0-99，输出域是0-2

​ 离散型，均匀分布的性质。 0 ，1， 2 差不多都是 33 个

输出和输入的规律无关 比如 陈广1 和陈广2 经过hash处理后 输出可能会差别特别大。 而且在整个输出域均匀分布

hash表的细节

有一个数组。比如大小是17.

有一个map key 是 A value 是 2

我先用key得到一个hash值，然后在对17取模就可以得到一个数字 假设是2。

我先看看 2 的位置有没有元素，没有。就直接挂在 2的位置

假设 还有一个map key 是A value 是 3；

去数组还是找2的位置。已经有元素的是A ，比对value的值， 如果不同，替换原来value的值，相当于修改操作。

如果这个数组里面没有A ，就加上一个A ,value 是3；

就是如果冲突了，就往上挂，一个链表而已。

数组上每个位置的元素都是均匀涨的。

比说我某个链的长度是3了，那么其它链的长度也差不多是3了

此时，我就进行hash表的扩容

为什么扩容呢？因为我觉得这个链表的长度已经是5了。在多的话会影响效率。

原来的数组大小是17.现在假设扩容后是104.

以前都是对17取模，现在对104取模。原来的就失效了。

所以就要把每个数据取出来重新用hash函数算一下，算完之后对104取模。重新分配他在哪一个桶里。

为什么能到达O（1）呢？

如果每次扩容的时候，都让长度扩大一倍，到n的时候要扩 logn的复杂度。问题是 我可以把容量扩大5倍，这样效率又要快。但还不是O（1）

实际上，还可以进行离线扩容

当链表已经到5了，我用的时候还用这个hash表，我在离线的时候进行扩容，当扩容结束后，在把所有的请求打到新的hash表上

因为有很多的优化技巧，所以很快，可以说是O（1）的

JVM中还是有一个桶，每一个桶里放一颗红黑树，是一颗平衡搜索二叉树。这样做出来的hash表

但是经典的结构是桶里面放链表

题1： 大文件

有一个大文件，每一行都有一个字符串，我想把重复的字符串打印出来

问：你给我多少台机器啊？

答：1000台

你把机器编号，从0 到999

问：文件存在哪里啊？

答：存在一个分布式系统上

问：我想处理这个大文件，按行读，又没有高效的工具啊？

答: 有，你就告诉我怎么处理把

我用这个高效的工具把每一行的数据读出来文本

利用hash函数生成一个hashCode

用hashCode对1000取模

如果是0，就把这条文本扔给0机器

如果是3，就把这条文本扔给3机器

如果是999，就把这条文本扔给999机器

这样一来就把大文本分到1000台机器上

而且相同的文本一定会分到一台机器上（相同的输入，一定会有相同的输出，取模后还是相同的）

这样的话，你有多少中不同的种类，就可以均匀的分布的1000台机器上。

如果分完之后每一个的大小还是太大，

你在每一台机器上把这个文件用hash函数分成小文件

然后每一台机器上可以跑并行任务

这样的话，就可以了。用hash函数做分流

hash函数相同输入相同输出，不同输入，均匀分布的特性

求完全二叉树节点的个数

发表于 2020-01-03 更新于 2020-01-06

要求 ： 时间复杂度小于O(n)

如果一棵树是满二叉树，高度是l，那么节点的个数是2的l次方-1

首先，先遍历二叉树的左边界，因为已经告诉你这是一颗完全二叉树了，你遍历出这个二叉树的左边界，就能求出这棵树的高度了，代价是logn的

然后遍历X右子树的左边界，右子树的左边界到没到最后一层，如果这个右子树的左边界到了最后一层，X的左子树就是满的，而且左子树的高度也能求出来 -1 就是3. 左树的节点数满足公式 2的3次方-1 + 当前节点 就是2的3次方

右树也是完全二叉树，递归求右树的节点个数，子问题和母问题等效

情况2：右子树的左边界不是L-1,也就是左子树也不一定满，但是根据定义，右子树一定是满的，就可以求出右子树的节点个数-2；

判断是否是完全二叉树

发表于 2020-01-03 更新于 2020-01-07

完全二叉树是 一颗二叉树 满足 上面的层都是满的，新的层都是从左向右的。

评判标准1：有右孩子没有左孩子，就不行

我依次拿到的1，2，3，4，5，6，7，8，9，在这个过程中任何一个节点，有右孩子没有左孩子，直接返回false

评判标准2 ：左右两个孩子不全

boolean flag = false;在2之前都为false，满足2的条件后就为true，代表一个阶段，后面遇到的所有节点，都必须是叶子节点

例如：

遇到1，中情况1了没（有右没左）？没有

中情况2了么（左右两个孩子不全）？ 没有

来到2，来到3，来到4，

来到5，中情况1了没（有右没左）？没有

中情况2了么（左右两个孩子不全）？ 中了哈

所以5以后的节点，6，7，8，9，10都必须是叶子节点，否则返回false
