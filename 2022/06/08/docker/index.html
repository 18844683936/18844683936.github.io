文件的赋值和目录操作

往镜像里复制文件有两种方式，COPY和ADD

复制普通文件

COPY和ADD都可以把local的文件复制到镜像里，如果目标目录不存在，则会自动创建

可执行文件的权限也会复制到image中
比如把本地的hello.py复制到/app目录下，/app这个目录不存在，则会自动创建

复制压缩文件

ADD比COPY高级一点的地方就是，如果复制的是一个gzip的压缩文件时，ADD会帮助我们自动去解压缩文件。

如何选择

所有的文件复制均使用COPY指令，仅在需要自动解压缩的场合使用ADD

WORKDIR

WORKDIR /app 相当于 cd /app 不同在于，如果 app目录不存在，则自动创建

构建参数和环境变量（ARG vs ENV）

CMD容器启动命令

CMD可以用来设置容器启动时默认会执行的命令。

  • 容器启动默认执行的命令
  • 如果docker container run 启动容器时制定了其他命令，则CMD命令会被忽略
  • 如果定义了多个CMD，只有最后一个会被执行
    如果Dockerfile中没有设置CMD，默认会进入到shell中（基础镜像中有）

    容器清理命令

    容器启动命令ENTRYPOINT

    ENTRYPOINT 也可以说设置容器启动时要执行的命令，但是和CMD 是有区别的。
    CMD设置的命令，可以在dcoker container run 时传入其他命令，覆盖掉CMD的命令，但是ENTRYPOINT所设置的命令是一定会被执行的
    ENTRYPOINT 和 CMD 可以联合使用，ENTRYPOINT设置执行的命令，CMD传递参数

    ##Docker的存储
    默认情况下,在运行中的容器里创建的文件，被保存在一个可写的容器层：
  • 如果容器被删除了，则数据也没有了
  • 这个可写的容器层是和特定的容器绑定的，也就是这些数据无法方便的和其他容器共享
    Docker 主要提供了两种方式做数据的持久化
  • Data Volume，由Docker管理，(/var/lib/docker/volumes/Linux),持久化数据的最好方式
  • Bind Mount，由用户指定存储的数据具体mount在系统什么位置

    Data Volume

    容器stop和start并不会删除原有的数据，如果删除容器，数据会丢失 Mountpoint就是Dockers Volume 所存储的位置， 默认删除容器后新添加容器，会重新生成一个新的 volume，这不是我们想要的，我们想要的是能够继续使用之前的那个 volume

Docker的网络

###查看容器的IP地址

###容器间通信之bridge模式

  1. 如果两台电脑之间通信，最简单的方法就是拿一根网线来给他们连接起来
  2. 如果有很多台电脑，就不能相互连接网线了，就需要连接到交换机，路由器，使用DHCP自动分配ip 输入ip a 后 会看到一个docker0 这就是Linux的bridge 网桥 他们都同时连接到一个默认的docker的Linux bridge上了，叫做 docker0 ![image-20220615150130548](https://blog-1305043867.cos.ap-shenzhen-fsi.myqcloud.com/undefined/202206/15/150131-686352.png) `brctl`使用前需要安装，可以通过`yum install -y bridge-utils`安装，对于Ubuntu,可以通过`sudo apt-get install -y bridge-utils` 可以显示Linux bride 相关的信息 ### 容器对外通信之bridge模式 为什么容器内部可以ping 通百度呢？ 容器的ip地址和机器的ip地址并不相同，请求出去之前会做一个NAT转换，会转换成10.158.0.1出去 ### 网络知识补充NAT ### 创建和使用自定义的bridge 创建容器的时候可以指定使用的网络，默认情况下它回去连接docker0,使用 --network参数来设置 一个容器也可以配置两个bridge `docker network connect bridge box3` 查看容器就会看到有两个network ![image-20220620090249953](https://blog-1305043867.cos.ap-shenzhen-fsi.myqcloud.com/undefined/202206/20/090250-952835.png) 进入该容器中查看ip `docker container exec -it box3 sh` ![image-20220620090645831](https://blog-1305043867.cos.ap-shenzhen-fsi.myqcloud.com/undefined/202206/20/090647-946678.png) 也可以断开容器和bridge `docker network disconnect bridge box3` 在连接相同自定义bridge的容器中，可以直接使用名称来ping通，例如 `ping box3` 创建bridge的时候可以自定义gateway和subnet ### 容器的端口转发 第一个端口是外部端口，第二个端口是容器的端口 ### host网络 使用host网络就相当于与主机使用同一个网络， 就相当于在本地启动了这个程序。
