前端布局

布局中的尺寸与位置

简介

  1. 尺寸与位置
    • 多个尺寸元素排列成一行或者一列就组成了位置
  2. 常见尺寸
    • 长方形，正方形，两个互相嵌套的，根据内容多少进行适配
  3. 常见位置
    • 一行，一列，还有行中嵌套列的，就像一个列表

CSS盒模型的组成

  1. 在CSS中，所有的元素都被一个个 盒子 包围着
  2. 盒子的组成：content内容，padding内填充，border边框，margin外边距
    • content 就相当于快递的物品
    • padding相当于填充物
    • border相当于盒子
    • margin就是多个快递之间的间距
  4. CSS 盒模型的注意点
    1. padding不能为负值，而margin可以为复制，margin成为负数后会叠加
      • 上面就是第一个粉色盒子的margin为负数的效果
    2. 背景色会平铺到非margin的区域 content+padding+border
    3. margin-top传递的现象
      • 给蓝色盒子添加margin-top属性的话，就会变成下面这样，并没有和粉色盒子产生间距，而是将粉色盒子一起带下来
      • 可以给父元素加上边框来解决这个问题
    4. margin 上下叠加的现象
      • 可以只给一个盒子添加magin属性

块级盒子与内联盒子

在CSS中常常使用两种盒子 - 块级盒子和 内联盒子

  1. 块级盒子有 div p h1 … 内敛盒子有span a strong …
  2. 块级盒子的特性
    1. 独占一行
    2. 支持所有css样式
    3. 不写宽度的时候，会和父元素的宽度相同
    4. 所占据的区域是一个矩形
  3. 内联盒子的特性
    1. 盒子不会产生换行
    2. 有些样式不支持，width，heigth，maring和padding都有问题
    3. 宽度由内容决定
    4. 所占的区域不一定是矩形
    5. 两个内联盒子之间由间隙，可以用将两个内联盒子挨在一起来解决
    6. 布局排版不要用内联盒子，写一些文本标签，修饰标签的时候用内联标签

自适应盒子模型的特性

自适应盒模型指的是，当盒子不设置宽度时，盒模型相关组成部分的处理方式是如何的

  2. 蓝色盒子是再红色盒子内部的
  3. 但是，如果蓝色盒子没有width属性，添加上padding等属性，整个盒子也不会变大，会反向所需content的区域
  4. 容器和父容器相同，并且内容是向里缩进的， 就可以使用这个特性

标准盒模型与怪异盒模型

  1. 在标准盒模型中，如果你给盒子设置heigth和width，实际上设置的是content box。padding 和border加上设置的宽高一起决定整个盒子的大小

  2. 在怪异模型中，所有宽度都是可见宽度，所以内容宽度是该宽度减去边框和填充部分

  3. box-sizing属性

    1. content-box : 标准盒模型， width和height来决content
    2. border-box: 怪异盒模型。width,heitht 决定 content+padding+border

  4. 应用1: 量取尺寸时不用再去计算一些值

  5. 应用2: 解决一些需要设置百分比和盒模型值

浮动样式详解

当元素被浮动时，会脱离文档流，根据float值向左或向右移动，知道他的外边界碰到父元素的内边界或者另一个浮动元素的外边界为止，是CSS中实现左右布局的一种方式

  1. 文档流:文档流是元素在Web页面上的一种呈现方式，按照出现的先后顺序进行排列

  5. 清除浮动的方法

    1. clear:right;clear:both

    2. BFC:清除高度塌陷的问 题

    3. 缺点：每次都得写一个空标签

    4. 使用after 伪类

浮动特性注意点

  1. 浮动的元素只会影响后面的元素

  2. 文本不会被浮动元素覆盖 ：红色盒子里面的文本没有被覆盖（初衷是来设计环绕文本的）

  1. 具备内联盒子的特性，宽度由内容决定，加上浮动后具备块级盒子特性：支持所有样式

  2. 浮动元素太多，一行放不下，会自动换行

定位样式详解

  1. CSS position属性用于指定一个元素在文档中的定位方式，其中top，right，bottom，和left属性则决定了该元素的最终位置
  2. 相对定位及特性
    1. 相对定位的元素是在文档中的正常位置偏移给定的值
    2. 不影响到其他元素布局
    3. 相对于自身进行偏移
      • 坐标（0，0）在方块的左上角
  3. 绝对定位及特性
    1. 会脱离文档流，不占据空间
    2. 具备内联盒子的特性，宽度由内容决定
    3. 具备块级盒子所有特性，支持所有属性
    4. 绝对定位元素相对于最近的非static祖先元素定位。当这样的祖先元素不存在时，则相对于可视区定位

固定定位特性注意点

  1. 固定定位与绝对定位相似，但是会固定在可视区中
  2. 具备内联盒子特性：宽度有内容决定
  3. 具备块级盒子特性：支持所有样式
  4. 固定定位元素不受祖先元素影响

总结

flex弹性布局

弹性盒子是一种用于按行或按列布局元素的一维布局方法。元素可以膨胀以填充额外的空间，收缩以适应更小的空间

主轴与交叉轴

弹性布局是需要一个容器的，也就是 flex container，除了容器还需要一个flex item 子项，可以发现，flex子项是默认水平排列的，水平排列的这条水平线就是主轴，main axis，在主轴的起点位置，就是主轴起点 main start，主轴的终点也就是 main end，主轴是水平的，并且是从左向右排列的

cross axis 是交叉轴，cross start 就是起始，cross end 就是结束位置，交叉轴是垂直的，并且是从上到下的顺序

  5. 如果一行有两个子盒子，一个盒子是1，另一个盒子是2，则第一个盒子会拓展剩余空间的1/3，第二个盒子会拓展剩余空间的2/3; 如果一个盒子是0.1，一个盒子是0.2，则第一个盒子占剩余空间的0.1…

flex-shrink 收缩比例

  1. 默认值是1，表示flex容器空间不足时，元素超出的部分全部收缩不见
  2. 值为0，表示flex容器空间不足时，元素超出的部分全部都不收缩，完全显示
  3. 值为0.5，表示flex容器空间不足时，元素超出的部分，只显示一半

flex-basis 为主轴方向设置长度，优先级大于widht和height

  1. 值为0：表示该长度为0，如果有值，就是最小长度，会换行
  2. 值为auto：刚好把该方向的文字全部包裹
  3. 值为200px：再该方向的长度为 200px
  4. 值为100%，该方向全部填满

flex 缩写

  1. flex属性是 flex-grow，flex-shrink和flex-basis的缩写

Grid 网格布局

章节介绍

  1. CSS网格是一个用于Web的二维布局系统。利用网格，你可以把内容按照行与列的格式进行排版。另外，网格还能非常轻松地实现一些复杂的布局
  2. 网页中经常充斥着行列组合的布局，比如说: 单元格的合并，格子和格子之间的间隙

定义网格及fr单位

  1. 网格布局就像Excel一样有行和列，Row来表示行，Column来表示列，一个单元格叫Cell，间隙叫Gap,单元格是可以合并的，合并完之后是一个Area

  2. 定义行和列分别用的是grid-template-rows和grid-template-colums，基于网格行和列的维度，去定义网格线的名称和网格轨道的尺寸大小

  3. 单位fr表示比例

    1. grid-template-rows: 1fr 1fr ; 表示分成两行，平均分
    2. grid-template-rows: 2fr 1fr ;表示分成两行，第一行是第二行的两倍宽
    3. grid-template-rows: 0.3fr 0.3fr ;表示分两行，每一行只有整个容器的0.3宽度,还会剩下0.4的容器

合并网格及网格命名

  1. grid-template-areas： 使用命名方式定义网格区域，需配合grid-area属性进行使用（还可以基于线的方式）
  4. grid-template 简写

网格间隙及简写

  1. 用来设置元素行列之间的间隙大小，推荐使用row-gap，column-gap，gap，这几个也可以适用于弹性布局中

网格对齐方式及其简写

  1. justify-items, align-items,place-items：align-items—justify-items: 子项在自己的格子里面的对齐方式
  2. justify-content,align-content,place-content: 在整个容器比子容器加起来还大的情况下，那几个格子整体的对齐方式
