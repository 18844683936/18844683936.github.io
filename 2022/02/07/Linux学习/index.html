初级篇

浏览和切换目录

  1. 浏览目录

    • ls 查看当前目录下所有的文件

    • ls - a 显示所有文件，包含隐藏文件

    • ls - l 已列表的方式来显示文件

    • ls - h 显示文件的大小单位，kb起步

    • ls - t 显示按照最近一次修改排序 t代表时间

  2. 切换目录 cd

    • cd 目录路径 跳转到指定目录下面
    • cd / 跳转到根目录
    • cd ../.. 跳转到爷爷目录
    • cd ~ （或者直接cd）跳转到家目录
    • du 统计当前目录的大小

操纵文件

  1. cat和less命令： 显示文件内容

    cat命令可以显示该文件的所有内容 ，可以添加两个文件，把这两个文件连接起来一并显示，参数是 -n 表示显示行号，适合查看小一点的文件

    • cat -n Xorg.0.log Xorg.9.log

    less命令可以一页一页的查看文件内容，快捷键有：

    • 空格键 ： 文件内容读取下一个终端屏幕的行数，相当于PgDn键
    • 回车键： 往后翻一行
    • d键：往后翻半页
    • b键 ：往前翻一页，相当于PgUp键
    • y键: 往前翻一行
    • u键：往上翻半页
    • q键: 停止less命令
    • = 键：显示你在文件的那个位置
    • / 键: 查找文件 ，配合n键，定位到下一个，N 定位到上一个

  2. head命令和tail命令

    head用于显示文件的开头几行，tail命令用于显示文件的末尾几行

  3. touch 命令

    • touch 命令本来是用于修改文件的时间戳的 ，让电脑以为目前刚创建的文件

    • 如果文件名中包含空格就用“”来扩住，

    • 可以同时创建多个文件

    • 用于创建一个目录

    • 可以同时创建多个目录

文件的复制和移动操作

  1. cp和mv命令，拷贝文件和移动文件

    1. cp命令：拷贝文件或目录

      • cp可以拷贝多个文件，也可以拷贝目录

      • 不一定要在同一目录下拷贝，也可以把文件拷贝到其他目录

      • 只需要把cp命令的第二个参数换成目录名：

      • 如果需要拷贝到目录下并修改文件名称的话：

    2. cp命令：拷贝目录

      • 只要在cp命令之后加上 -r 或者-R参数, r 大小写作用一样，都表示递归，recursive

      • *号代表匹配所有的字符

    3. mv命令:移动文件或目录

      • mv 移动文件或目录

    4. mv命令：重命名文件

      • mv命令可以重命名文件，还得归因于它的机制：移动文件 ，经过mv移动之后，原始文件变成了新的名字的文件

    5. rm命令：删除文件和目录

      • rm可以同时删除多个文件，用空格隔开每个文件即可

      • rm命令可以添加上-i参数，用来确认是否删除该文件，i 代表 inform ，表示告知，通知

      • rm 命令可以添加 -f 参数，强制删除

      • rm -r 递归的删除，用来删除目录

      • rmdir 只能删除空目录

      • rm -r 和 -f 参数： 极为危险

    6. ln命令：创建链接

      • ln是link的缩写，表示链接

      • ln命令用于在文件之间创建链接，类似于快捷方式

      • Physical link : 物理链接或硬链接

        1. 硬链接原理: 使链接的两个文件共享同样的文件内容，就是同样的innode

        2. 一旦文件1和文件2之间有了硬链接，那么你修改文件1或文件2，修改的是相同的一块内容

        3. 只能创建只想文件的硬链接，不能创建指向目录的

        使用ls -l 来看到这个3 代表有3 个指向同一个inode的链接

        1. 如果我们用rm file1 来删除file1 ，对file2也没啥影响，对于硬链接来说，删除任意一方的文件，共同指向的文件内容并不会从硬盘上被删除，只有既删除file1又删除file2 他们共同指向的文件内容才会消失，也就是那个inode才会被删除

      • Symbolic link : 符号链接或软链接

        1. 软链接才真正像Windows下的快捷方式

        2. 创建软连接需要添加 -s 参数，是symbolic 符号的缩写

        3. 如果删除file2 链接文件的话，file1不会有任何影响，如果删除file1源文件的话，file2就会变成一个死链接

        4. 软链接可以指向目录，但是硬链接是不行的

用户和权限

  1. sudo命令：以Root身份运行命令

    • sudo 命令，暂时成为root用户，是Substitute（替换） User Do 的缩写 sudo + cmd

    • sudo su 命令； 一直成为root 用户

    • exit 命令，退出管理员身份，或者ctrl + D

    • exit 后，在输入 su 就会提示输入root用户的密码，获取root用户身份，但是所在目录还是普通用户的目录

    • 直接输入 su - 命令输入root密码之后也可以切换为root用户身份，而且会直接定位到 root的家目录

  2. 用户管理的命令（只能root权限才能执行）

    • useradd命令：添加新用户，但是还不能使用，因为没有分配密码
    • passwd命令可以修改用户的密码
    • userdel 命令可以用来删除用户，但是会遗留 家目录 可以添加 -r 或者 – remove 参数来 ‘抄家’

群组的管理和文件权限管理

  1. Linux中每一个用户都属于一个特定的群组，如果你不设置用户的群组，默认会创建一个和它的用户名一样的群组，并且把用户划归到这个群组

  2. groupadd:创建群组

    • groupadd friends

  3. usermod 命令：修改用户账户

    • -l参数: 对用户重命名。 /home 中的用户家目录不变，需要手动修改

    • -g参数：修改用户所在群组

    • 用groups(“群组”)命令可以获知一个用户属于哪个，哪些群组

    • 一次性将一个用户添加到多个群组，使用-G参数即可。群组直接用，隔开

    • 使用-G，-g的话，会将用户从原来的群组中移除

    • 如果你不想离开原先的群组，又想加入新的群组，可以在-G参数（不能用g）的基础上加上 -a参数，a是append的缩写

    • groupdel 命令:删除群组

  4. 修改文件的所有者和群组

    • 只有root用户可以修改一个文件的所有者和群组

    • chown命令:改变文件的所有者，change owner

    • 第一个charles表示所有者，第二个charles表示群组

    • ```bash
      chown tomas file3 # 将file3的所有者改为tomas

      chgrp tomas file3# 将file3文件的群组改为tomas

    • -R参数：递归设置子目录和子文件，必须R，会使得被修改的目录的所有子目录和子文件都改变所有者，或者连群组也改变，用上述冒号的方法来同时修改所有者和群组

文件权限管理

  1. 权限的原理

    • 文件信息的第一列比较富足，可以看到不少 d,r,w,l,x等字母，可以称之为 文件访问权限符

    • d: 英语directory 缩写，就是说这是一个目录。

    • l: 英语link的缩写，表示链接，就是说这是一个链接

    • r，w，x表示可以读取，写，执行，x是excute的缩写

    • 如果x权限在一个目录上，那么表示这个目录可以被读

    • 如果相应位置有字符，表示有相应的权限。- 表示没有相应的权限

    • 为什么我们看到这一排有好多个重复出现的r，w和x呢？，那是因为访问权限是按照用户来划分的

  2. chmod 命令：修改文件的访问权限

    • chmod命令不需要是root用户就可以运行，只要你是此文件的所有者，就可以用chmod来修改文件的访问权限

    • 用数字来分配权限：chmod的绝对用法

      1. Linux系统对每种权限（r,w,t）分配了对应的数字

      3. 要合并这些权限，就需要做简单的加法了:将对应的数字相加

      4. 假如我们要分配读权限和写权限，那么我们就要用 4+2=6

      5. 访问权限的三组，所有者的权限，群组用户的权限，其他用户的权限，只要分别做加法就可以了，然后把三个和连起来

    • 用字母来分配权限：chmod的相对用法

      1. u:user的缩写，用户，表示所有者
      2. g：group的缩写，表示群组用户
      3. o:other的缩写，表示其他用户
      4. a:all 的缩写，是英语”所有”的意思。表示所有用户
      5. +：加号，表示添加权限
      6. -：减号，表示去除权限
      7. = ：等于号，表示分配权限
      11. chmod 配合-R 参数可以递归的修改文件

查找文件

  1. locate 命令，快速查找文件

  2. 文件数据库

    • 刚创建的文件，但是locate命令没有找到它
    • locate命令不会查找磁盘，会查找数据库，对于刚创建不久的文件，因为他们还没有被收录进文件数据库
    • Linux系统一般每天更新一次文件数据库，可以使用updatedb命令强制执行更新数据库 用管理员账户来操作

  3. find命令，深入查找

    • find命令会遍历硬盘

    • find 何处 何物 做什么

    • find命令可以限定查找目录。比如可以只让find查找./home 目录，默认的，假如没有给何处 这个参数，那么find命令会在当前目录以及子目录中查找

    • 何物 也就是要查找什么。我们可以根据文件的名字来查找，也可以根据其大小来查找，也可以根据其最近访问时间来查找，等等。这个参数是必须的

    • 做什么 用find命令找到我们要的文件后，可以对每个文件做一定的操作，称为后续处理 ，默认的，加入不指定这个参数，find命令只会显示找到的文件

    • **find -name “newFile” ** 用来查找当前目录下名为newFile的目录

    • find /var/log -name “syslog” 在var/log目录下查找名为syslog的文件

    • 不同于locate命令，find命令要想模糊搜索，得 用正则表达式

    • find /var -size +10M 查找在var目录中大小超过10M的文件

    • find /var -size -40k

    • 根据文件的最近访问时间查找

    1. 可以使用 -atime 参数。 atime 是access和time的缩写
    2. find -name “*.txt” -atime -7 7天内访问过的 .txt结尾的文件

    • find -name “newFile” -type d 查找文件类型是目录的相关文件

    • find -name “*.jpg” -delete 删除当前目录及子目录下所有以.jpg为后缀的文件

    • find one -name “*.txt” -exec chmod 600 {} ; 修改在one 目录下查到到txt文件的权限为600

    1. 这个操作不必用双引号括起来
    2. {} 会用查找到的每个文件来替换
    3. ; 是必须的结尾
    4. 如果你对没有确认提示不太放心
    5. 可以将-exec参数换成-ok参数，用法一样

正则表达式和数据操作

  1. grep 命令: 筛选数据

    1. grep 是全局搜索一个正则表达式，并且打印的意思
    2. 在文件中查找关键字，并显示关键字所在的行

  2. grep text file text代表要搜索的文本，file代表供搜索的文件

  3. grep path /etc/profile 在etc的profile文件中查找所有包含 path的行

    1. 默认grep命令是区分大小写的,比如我搜索的文本时text，就不会搜出Text

    2. 可以给grep加上-i参数，使其忽略大小写

    3. -n 参数: 显示行号

    4. -v参数: 只显示文本不在的行 就是只显示搜索文本不在的那些 行

    5. -r参数:在所有子目录和子文件中查找 grep-r “Hello World” folder/ 在folder目录的所有子目录和子文件中查找Hello World 这个文本

    6. -I参数 排除二进制文件

  4. grep的高级用法:配合正则表达式

    1. -E参数:使用正则表达式

      • grep -E ^path /etc/profile 匹配 行首（匹配输入字符串的开始位置）

      • grep-E [Pp]ath /etc/profile []表示可以匹配P和p

      • grep-E [0-4] /etc/profile 匹配出现0-4的所有的行

      • grep -E [a-zA-Z] /etc/profile 包含在az之间的任意字母或者A-Z之间的任意字母

  5. Sort命令:为文件排序。

    1. sort name.txt 将name文件中的字符按照A-Z排序，不区分大小写

    2. -o参数: 将排序后的内容写入新的文件 sort -o name_sorted.txt name.txt

    3. -r参数:倒序排列 sort -r name.txt

    4. -R参数: 随机排序 sort -R name.txt

    5. -n参数 : 对数字排序

      • 默认的,仅用sort命令的时候，是不区分字符是否是数字的，会把这些数字看成字符串，按照1-9的顺序来排，例如138会排在25前面，因为1排在2的前面

      • sort -n number.txt 按照大小排序

  6. wc 命令:文件的统计

    1. wc 是 word count的缩写，貌似是用来统计单词数目的，但是wc的功能不仅止于此，还可以用来统计行数，字符数，字节数等

    2. wc name.txt

      表示 有4行， 4个单词， 24个英语字符，24个字节，包含换行符

    3. -l 参数:统计行数 单独统计行数 line 的意思 wc -l name

    4. -w参数: 统计单词数 w是英文 word 的缩写 wc -w name.txt

    5. -c参数: 统计字符数

  7. uniq命令:删除文件中的重复内容

    1. uniq repet.txt

    2. uniq repeat.txt unique.txt 将去重后的文本放入unique.txt中

    3. -c参数 统计重复的行数 count unique -c repete.txt

    4. -d参数 只显示重复行的值 unique -d repete.txt

  8. cut命令: 剪切文件的一部分内容

    1. -c参数 c 是 character， cut -c 2-4 name.txt 每一行只显示第二个到第四个字符

    2. cut命令进阶:根据分隔符来剪切

      4. -d参数 d是 delimiter的缩写 ，用于指定用什么分隔符，比如逗号，分号，双引号等等。

      5. -f参数 : f 是field的缩写，表示 剪切下用分隔符分割的那一块或那几块区域

      6. cut -d , -f 1 notes.csv -d指定使用逗号分隔符， -f 1 表示取分隔下来的第一个，所以结果是:

        cut -d , -f 1,3 notes.csv -f 1,3 表示会截取第一部分和第三部分:

流，管道，重定向

  1. 输出重定向

    1. 把本来要显示在终端的命令结果，输送到别的地方，输送到别的地方或者作为其他命令的输入

    2. 如何实现：通过在命令间插入特定的符号，这些符号称为 重定向流 符号

    3. > 和 >> : 重定向到文件 就不会在终端显示命令运行结果了

      cut -d , -f 1 notes.csv > sutdents.txt 重定向到students.txt文件中

      1. 使用> 符号要小心，如果此文件不存在，则新建一个文件，如果此文件已经存在，那就会把文件内容覆盖掉而且是不会征求用户确认的

    4. 黑洞文件 /dev/null 如果你既不想在控制台显示，又不想将内容放到文档中，那就放到这里吧，/dev/null 文件时特殊文件，不是一个目录，它总是空的，它能使发送到 /dev/null的任何数据作废

    5. >> : 重定向到文件末尾

      1. >> 作用与 >是类似的，不过他不会像> 那么危险，而是将重定向的内容写到文件末尾，起到追加的作用
      2. 如果文件不存在，文件会被创建

    6. stdin ,stdout, stderr

      1. 从键盘向终端输入数据，这是标准输入，也就是stdin
      2. 终端接收键盘输入的命令，会产生两种输出

        • 标准错误输出：stderr 。 错误输出

    7. 2> 符号 标准错误输出的文件描述符是2，所以这里的2表示标准错误输出

      1. cat not_exist_file.csv > results.txt 2> errors.log 将错误信息写入到 errors.log 中

    8. 2>> 将错误信息追加到文件末尾

进程和系统监测

  1. w命令: 都有谁，在做什么

    1. 可以快速的查找系统中都有哪些用户登录着

    2. 第一行内容也可以用 uptime命令获知， 当前时间，系统运行了多久，多少用户，负载情 况

      表示一分钟之内的平均负载，5分钟之内的平均负载，15分钟之内的平均负载，单核的处理器负载超过1就过载了，双核的处理器负载超过2就过载了可以使用tload 来查看系统负载图

  2. ps命令:列出运行的进程

    1. ps 是Process Status 的缩写
    2. ps 命令显示的进程列表不会随时间而更新，是静态的，是一个进程的快照，snapshot
    4. PID 是 进程号的意思 process identifier 的缩写
    5. TTY 进程运行所在的终端 ， pts 伪终端从属
    6. CMD：产生这个进程的程序名，如果你看到好几行都是同样的程序名的话，那么就是同样的程序产生了不止一个进程
    7. ps显示的只是当前终端中的进程

  3. ps -ef 列出所有进程 ，

    1. 可以使ps命令列出所有用户在所有终端的所有进程
    3. UID: 用户名，运行进程的用户
    4. PPID：程序的父进程号
    5. ps -ef | less ：可以显示一部分

  4. ps -efH : 以乔木状态列出所有进程

  5. ps -u 用户名 ：列出此用户运行的进程

  6. ps -aux: 通过CPU和内存使用来过滤进程

    1. %CPU cpu的占比

    2. %MEM 内存的占比

    3. STAT 进程状态

    4. START 什么时候启动的

    5. TIME 当前运行多少时间

    6. COMMAND 运行这个进程的程序是什么

    7. **ps -aux | less ** 默认结果是未排好序的，可以通过 –sort 来排序

    8. ps -aux –sort -pcpu | less : 根据CPU使用率来降序排列

    9. **ps -aux –sort -pmem | less ** 根据内存使用率来降序排列

    10. ps -aux –sort -pcpu,+pmem | less 通过CPU和内存使用来过滤进程

  7. pstree 以树形结构显示进程

    1. ps- axjf 和 pstree 效果类似

管理前台后台进程

  1. 前台进程和后台进程
    1. 默认情况下，用户创建的进程都是前台进程
  2. & 符号：在后台运行进程
    1. cp file.txt file-copy.txt & 后台拷贝文件
    3. 第一行表示进程号是15479
    4. 第二行表示后台进程的任务已经完成
    5. & 符号虽然常用，但是却有一个不可忽视的问题：后台进程始终与终端相关联，一旦终端关闭或者用户登出，进程就会自动结束
  3. nohup 命令 使进程与终端分离
    1. 当用户注销或者网络断开时，终端会收到HUP（hangup 挂断的意思）信号从而关闭其所有子进程
    2. 可以使用nohup命令使命令不受hup信号的影响
    3. 有些非常耗时的行为，就用到了nohup命令了
  4. Ctrl+Z 转到后台，并暂停运行
    1. 进程转入到后台之后，会显示他在当前终端下的后台进程编号
  5. bg命令 使进程转到后台
    1. bg是英语background的意思， bg命令的作用是将命令转入后台运行，假如命令已经在后台，并且暂停着，bg命令会将其状态改为运行
    2. 不加任务参数，bg命令会默认作用于最近的一个后台进程，也就是刚才被Ctrl+Z暂停的top进程
    3. 如果后面加%1，%2这样的参数（不加%也行）则是作用于指定标号的进程。
  6. 5种常见的进程状态
    1. 运行 对应的状态码是R runtime的意思
    2. 中断 当某个条件形成后或接受到信号时，则脱机该状态 状态码是 S sleeping 的意思
    3. 不可中断 进程不响应系统异步信号，即使用kill命令也不能使其中断 对应的状态码是D
    4. 僵死 进程已经终止，但是进程描述符依然存在，直到父进程调用wait4()系统函数后将进程释放 状态码是 Z 表示 僵死 zombie
    5. 停止 进程收到SIGSTOP, SIGSTP, SIGTIN,SIGTOU 等停止信号后停止运行，状态码是T 表示 traced or stopped
  7. jobs命令: 显示后台进程状态
    1. 显示当前终端里的后台进程状态
  8. fg命令: 使进程转到前台，与bg命令正好相反
    1. 是 foreground的意思，表示前台
    2. 用法和bg类似

任务的定时和延期

  1. date命令 :调节时间
    1. **date “+%H” ** 当前几点了
    2. date “+%H:%M:%S” 当前时间
    4. date命令修改系统时间 (需要root用户)
  2. at命令：延时执行一个程序
    1. 可以用at命令来设定一个程序的执行时间，at是 在……时刻 的意思，at命令只能让程序执行一次
    2. 在指定时刻执行程序
      • 先用at命令后接想要执行程序的确定时刻
      • 在输入你想要在以上指定时刻执行的命令
      • 使用Ctrl+D 来结束at命令，at会显示EOT ，End Of Transmission
      • at 22:10 tomorrow 在明天的22点10分执行
      • at 22:10 12/10/19 在2019年12月10号的22点10分执行
    3. 在指定间隔之后执行程序
      • at now +10 minutes 从现在开始的10分钟后执行
    4. atq和atrm命令：列出和删除正在等待执行的at任务
      • atrm 接 编号任务 就能删除正在等待执行的任务
  3. sleep 命令：休息一会儿
    1. 在两句命令之间可以插入一定的暂停等待时间，使用sleep命令
    2. touch file.txt; sleep 10; rm file.txt
    4. 默认sleep 的单位是s ， 可以更换单位为， 例如sleep 15m 就是15分钟的意思
  4. &&和||符号
    1. 这两个符号和分号；一样，用于分隔两个命令，使得命令依次执行
    2. &&表示之前的命令执行成功，才会执行后面的命令
    3. ||表示之前的命令执行失败，才会执行后面的命令
    4. ；表示不论之前的命令怎么样，都会执行后面的命令
  5. crontab 命令：定时执行程序
    1. at命令只能执行某个或者某几个命令 一次，crontab却可以重复执行命令，例如:每小时，每分钟，每天，每星期等等
    2. 安装crontab
      • 把这句话加到 .bashrc 文件中来修改默认文本编辑器export EDITOR = nano
      • echo “export EDITOR=nano” >> ~/.bashrc 向该文件追加
      • source .bashrc 使配置立即生效
    3. -l参数 显示crontab文件
    4. -e参数 修改crontab文件
      • 编辑的格式：m h dom mon dow command ，给出了crontab中的每行指令的书写格式
      • 10 22 * * * touch ~/file.txt 每天22点10分在家目录创建file.txt
    5. -r参数 删除crontab文件
      • 删除最近创建一次的crontab

对称加密和非对称加密

对称加密

  1. 对称加密算法用一个密钥来加密信息，密钥是一个参数
  4. 对称的意思就是指 ，加密和解密使用的是同一个密钥
  5. 缺陷：必须谨慎的传递密钥
  6. 为了加密用于对称加密的密钥：将用另一种方法：非对称加密

非对称加密

  1. 一个是 公钥 用于加密
  2. 一个是私钥 用于解密
  3. 公钥只用来加密
  4. 用非对称加密的算法，我们就只能用私钥来解密
  7. 公钥可以在网络上以明文传输，毕竟是公开的密钥
  8. 最有名的是RSA算法
  9. 缺陷：非对称加密比对称加密慢100-1000倍

SSH 创建一个安全的通信管道

  1. 首先，使用 非对称加密 ，安全的传输 对称加密 的密钥
  2. 之后，就一直使用对称加密的密钥来作为加密和解密的手段
  3. 非对称加密只是在通信之初用于交换对称加密的密匙
