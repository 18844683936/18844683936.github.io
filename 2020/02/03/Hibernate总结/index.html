多表查询

  1. 一对多

    • 一的一方

      1. javaBeans：private HashSet<Role> roles = new HashSet<>();

      ​ 在加上getset方法，注意要实例化,注意用Set接口来接，不能用HashSet，否则会报错。

      1. hbm.xml：

    • 多的一方

      1. javaBean： private User user; 再加上getset，不需要实例化

      2. hbm.xml中加入多方的维护关系

    • 级联保存

      • 在一的一方设置级联保存（常用）

        1. 在一的一方的set上面添加属性，cascade="save-update"

        2. cg.getRoles().add(role);
cg.getRoles().add(role1);
session.save(cg);

        3. 保存多的一方的时候把多的一方也保存了

      • 双方都设置级联保存

        1. 保存任何一个都会全部保存

    • 级联删除

      • 不设置级联删除，删除一方，多方的外键为空

      • 在一的一方设置级联删除 cascade="delete"，删除一方，多方也删除。

      • 孤儿删除，在一的一方设置孤儿删除，cascade="delete-orphan"，主类中取消关系后，外键类自动删除

    • 级联的取值：

      • none ：不使用级联
      • save-update ： 级联保存
      • delete ： 级联删除
      • delete-orphan ：孤儿删除
      • all ： save-update + delete
      • all-delete-orphan : 全部
      • 为了防止产生多余的sql语句，那么需要一方来放弃外键的维护！多方不能放弃外键维护。 在<set>标签上配置一个inverse=”true”

  2. 多对多

    • 首先每一方的xml都是这样的：而且在其中一方设置级联保存，这样

    • 而且每一方的javaBean 中都有一个Set集合来维护多对多的关系

    • 级联删除

      • 在role角色方设置级联删除，相当于删除角色时，关联到该角色中间表数据被删掉，中间表数据关联的用户被删掉。删掉的用户在中间表的数据也被删掉

    • 总结：

      • 在多对多关系当中：必须有一方设置放弃外键维护，也就是说有一个hbm.xml文件set标签中设置inverse=“true”

        在一对多关系当中：一般 是一方放弃外键维护，如果不放弃多执行update。

   **在哪一方设置级联保存，save那一方的对象，关联的对象也被保存。**

   

   **在多对多关系中，在某一方设置级联删除，中间表外键为该记录的数据会被删除，另一方表中的关联中间表删除数据的数据也被删除。多对多关系慎用级联删除。**

  • 解除关系

    • 解除关系只会删掉第三张表的数据
